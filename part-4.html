Bagian IV: Keamanan Perangkat Lunak & Platform

Pengembangan aman, kerentanan web/mobile, dan daur hidup perangkat lunak

14Keamanan Perangkat Lunak

Ikhtisar

Kerentanan tingkat implementasi dalam perangkat lunak dan teknik untuk mencegah dan mengatasinya. Mencakup masalah keamanan memori, cacat injeksi, dan perlindungan runtime.

Topik Utama
  • Kerentanan Keamanan Memori: Buffer overflow, use-after-free, korosi heap
  • Cacat Injeksi: Injeksi SQL, injeksi perintah, injeksi kode
  • Kondisi Race: Kerentanan waktu, TOCTOU (Time-of-check-time-of-use)
  • Cacat Logika: Otentikasi rusak, bypass otorisasi
  • Analisis Statis: Pemindaian kode sumber, alat SAST
  • Pengujian Dinamis: Fuzzing, symbolic execution, monitoring runtime
  • Perlindungan Runtime: ASLR, DEP/NX, stack canaries, Control Flow Guard
  • Praktik Pengkodean Aman: Validasi input, pengkodean output, pustaka aman
Sumber: CyBOK v1.1.0 - Keamanan Perangkat Lunak. Hak Cipta Mahkota 2021.

15Keamanan Web & Mobile

Ikhtisar

Tantangan keamanan khusus aplikasi web dan platform mobile, mencakup kerentanan sisi klien dan server.

Topik Utama
  • Kerentanan Klien Web: Cross-Site Scripting (XSS), clickjacking, phishing
  • Kerentanan Server Web: Injeksi SQL, CSRF, deserialisasi tidak aman
  • Keamanan Browser: Same-Origin Policy (SOP), Content Security Policy (CSP)
  • HTTPS & TLS: Validasi sertifikat, pinning, konten campuran
  • Otentikasi: OAuth, OpenID Connect, WebAuthn, pengelola kata sandi
  • Manajemen Sesi: Cookie, token, pembajakan sesi
  • Platform Mobile: Model keamanan iOS dan Android, izin aplikasi
  • Ancaman Mobile: Malware, reverse engineering, exfiltrasi data
  • API & Microservices: Keamanan API, pembatasan laju, otentikasi
Sumber: CyBOK v1.1.0 - Keamanan Web & Mobile. Hak Cipta Mahkota 2021.

16Daur Hidup Perangkat Lunak Aman

Ikhtisar

Mengintegrasikan keamanan di seluruh daur hidup pengembangan perangkat lunak dari desain hingga penyebaran dan pemeliharaan.

Topik Utama
  • Model Daur Hidup: Microsoft SDL, SAFECode, kerangka Touchpoint
  • Desain Aman: Pemodelan ancaman, review arsitektur, pola keamanan
  • Pengembangan Aman: Standar pengkodean aman, review kode, pair programming
  • Pengujian & Verifikasi: Penetration testing, pengujian keamanan, integrasi CI/CD
  • Agile & DevSecOps: Keamanan berkelanjutan dalam pengembangan cepat
  • Risiko Pihak Ketiga: Manajemen dependensi, keamanan rantai pasokan
  • Penyebaran & Pemeliharaan: Konfigurasi aman, patching, update
  • Model Kematangan: SAMM, BSIMM, Common Criteria
  • Domain Spesifik: Cloud, IoT, sistem embedded, kendaraan otonom
Sumber: CyBOK v1.1.0 - Daur Hidup Perangkat Lunak Aman. Hak Cipta Mahkota 2021.
